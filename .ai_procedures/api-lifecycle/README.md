# API Lifecycle

## Overview

This folder contains procedures for API design, implementation, testing, documentation, and evolution. These procedures ensure APIs are well-designed, secure, and maintainable.

## Procedures

- **API Discovery Procedure** - Discovering existing APIs
- **API Design Procedure** - Designing APIs with OpenAPI/Types
- **Backwards Compatibility Procedure** - Ensuring API compatibility
- **API Implementation Procedure** - Implementing APIs
- **API Security Procedure** - Securing APIs
- **API Testing Procedure** - Testing APIs
- **API Documentation Procedure** - Documenting APIs
- **API Deprecation Procedure** - Deprecating APIs

## Usage

Follow these procedures when:
- Designing new APIs
- Modifying existing APIs
- Documenting APIs
- Deprecating APIs

## Related Lifecycles

- **Feature Lifecycle** - Features often include API changes
- **PR Lifecycle** - API changes go through PR review
- **Data Lifecycle** - APIs often interact with data stores
