# Change Management Lifecycle

## Overview

This folder contains procedures for change approval, tracking, and communication. These procedures ensure changes are properly documented, approved, and communicated.

## Procedures

- **Change Approval Procedure** - Process for obtaining change approval
- **Change Tracking Procedure** - How to track and document changes
- **Change Communication Procedure** - Communicating changes to stakeholders

## Usage

Follow these procedures when:
- Changes require formal approval
- Tracking change requests
- Communicating changes to stakeholders

## Related Lifecycles

- **Release Lifecycle** - Releases require change approval
- **Feature Lifecycle** - Large features may require change approval
- **Incident Lifecycle** - Incident fixes may require change approval
