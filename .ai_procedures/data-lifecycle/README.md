# Data Lifecycle

## Overview

This folder contains procedures for managing data across DynamoDB and Postgres. These procedures cover schema changes, migrations, data consistency, and data governance.

## Procedures

- **Data Classification Procedure** - Classifying data by sensitivity and type
- **Data Ownership & Source-of-Truth Procedure** - Defining data ownership and sources of truth
- **Schema Change Procedure** - Postgres migration procedures
- **Shape Evolution Procedure** - DynamoDB item versioning and evolution
- **Backfill / Reindex Procedure** - Procedures for data backfills and reindexing
- **Data Consistency Procedure** - Cross-store synchronization rules
- **Query/Index Review Procedure** - Reviewing queries and indexes
- **Backup & Restore Procedure** - Backup and restore procedures
- **Data Retention & Deletion Procedure** - Data retention policies and deletion procedures
- **Audit Logging Procedure** - Audit logging for data access

## Usage

Follow these procedures when:
- Modifying database schemas
- Performing data migrations
- Managing data across stores
- Implementing data policies

## Related Lifecycles

- **Feature Lifecycle** - Features often require data changes
- **API Lifecycle** - API changes may require data changes
- **Governance Compliance** - Data procedures support compliance requirements
