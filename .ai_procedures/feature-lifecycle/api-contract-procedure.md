# Procedure: API Contract

**Status:** Alias - Use [API Design Procedure](../api-lifecycle/api-design-procedure.md)

## Purpose

This procedure defines API endpoint definitions, request/response schemas, and OpenAPI specifications.

**Note:** This procedure is an alias for the [API Design Procedure](../api-lifecycle/api-design-procedure.md). The API Design Procedure creates the API contract (OpenAPI specification and type definitions) that all implementation layers depend on.

## Related Procedures

- **Prerequisite for:** Frontend Implementation Procedure, Backend Implementation Procedure, Middleware Implementation Procedure
- **Depends on:** [API Discovery Procedure](../api-lifecycle/api-discovery-procedure.md) (MUST discover existing APIs first), Architecture Review Procedure, Data Design Procedure (if applicable)
- **Used by:** Feature Implementation Orchestration Procedure
- **Critical:** All three implementation layers depend on this
- **See:** [API Design Procedure](../api-lifecycle/api-design-procedure.md) for complete procedure
