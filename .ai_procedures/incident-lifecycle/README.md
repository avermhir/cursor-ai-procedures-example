# Incident Lifecycle

## Overview

This folder contains procedures for responding to and resolving incidents. These procedures guide the team through incident triage, mitigation, communication, and post-incident activities.

## Procedures

- [Triage Procedure](./triage-procedure.md) - Initial incident assessment and prioritization
- [Mitigation Procedure](./mitigation-procedure.md) - Steps to resolve or contain the incident
- [Communications Procedure](./communications-procedure.md) - How to communicate incident status
- [Root Cause Analysis Procedure](./root-cause-analysis-procedure.md) - Analyzing the underlying cause
- [Corrective Actions Procedure](./corrective-actions-procedure.md) - Actions to prevent recurrence
- [Postmortem Procedure](./postmortem-procedure.md) - Documenting and learning from incidents

## Usage

Follow these procedures when:
- A production incident occurs
- System outages or degradation
- Critical bugs in production
- Security incidents

## Related Lifecycles

- **Release Lifecycle** - Incidents may trigger rollbacks
- **Feature Lifecycle** - Incidents may require hotfixes
- **Security Lifecycle** - Security incidents follow security procedures
- **Change Management Lifecycle** - Incident fixes may require change approval
