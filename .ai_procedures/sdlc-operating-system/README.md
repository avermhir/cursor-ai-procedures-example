# SDLC Operating System

## Overview

This folder contains core SDLC standards and practices that apply across all work in the organization. These procedures define the foundational practices that every team member should follow.

## Procedures

1. **Repository Standards Procedure** - Standards for repository structure, organization, and conventions
2. **Branching & Naming Procedure** - Git branching strategy and naming conventions
3. **Definition of Done Procedure** - Criteria that must be met before work is considered complete
4. **Code Review Standards Procedure** - Standards and best practices for code reviews
5. **Documentation Standards Procedure** - Requirements for documentation across the codebase
6. **Security Standards Procedure** - Security practices and requirements
7. **Testing Standards Procedure** - Testing requirements and standards
8. **Observability Standards Procedure** - Logging, metrics, and observability requirements
9. **Dependency Management Procedure** - How to manage and update dependencies
10. **Environment & Config Procedure** - Environment variable and configuration management
11. **IaC Standards Procedure** - Infrastructure as Code standards and practices
12. **Data Standards Procedure** - Data modeling, storage, and access standards

## Usage

These procedures are referenced by other lifecycle procedures but can also be used standalone when updating standards or practices.

**Related Lifecycles:**
- Feature Lifecycle (uses these standards during development)
- PR Lifecycle (validates against these standards)
- Release Lifecycle (ensures standards are met before release)
